DroidBotRecorder

I have been working on DroidBotRecorder, as a part of Google Summer of Code 2018. It is an interaction recorder created to aid semi-automated testing using DroidBot.

Introduction

Most (if not all) automated testing programs have the inherent limitation in that they cannot perform all aspects of testing, due to conditions like a login wall, or complex interactions that cannot be generated in an automated way.

DroidBotRecorder solves this problem by allowing the tester to semi-automate the test. It aims to record interactions and allow the user to inject these interactions automatically during an automated test.

Project Details

The project will require the following features:

  • A front-end to capture interactions over a browser. It includes the following:

    • Capturing mouse-events using JavaScript, and sending them to the server.
    • Capturing keyboard input, and sending that to the server.
    • Displaying the emulator screen on the webpage
    • Additional input such as hardware buttons, sensor input, etc. (for later)
    • Improved UI/UX

  • A middleware to capture and inject the inputs from the front-end, and stream the device display to the front-end. It needs:

    • A webserver to display the webpage
    • Logic to inject the events to the android device
    • A component to stream the device screen to a webpage

  • An analyzer that constructs UI interactions from traces. It needs:

    • Logging the input events to a file.
    • Constructing a UI tree for every screen that is encountered.
    • Correlating input to the UI.
